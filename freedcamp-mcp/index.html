Why this fork exists

Freedcamp's API is deep. The original MCP only touched the surface.

The upstream server was locked to a single project and focused on tasks. This fork opens up every project and adds the write tools Claude was missing — projects, comments, and bulk-safe operations.

Upstream — one project

One project. Tasks only.

Pinned to a single FREEDCAMP_PROJECT_ID, with no way to discover or manage anything beyond tasks.

  • Locked to one project at a time
  • No project or comment tools
  • First 200 items, silently truncated
This fork — whole workspace

Every project. Full lifecycle.

A per-call project_id always wins, plus project and comment write tools turn Claude into a real Freedcamp operator.

  • Works across every accessible project
  • Adds project + comment write tools
  • Pagination-aware & bulk-safe

Up and running

Point Claude at your Freedcamp.

Installed from source — it's a fork, so npx freedcamp-mcp would fetch the upstream. Clone, build, and register with your API key + secret.

bash — setup
# clone & build the fork
$ git clone https://github.com/justadityaraj/freedcamp-mcp.git
$ cd freedcamp-mcp && npm install && npm run build

# register with Claude (credentials in env)
$ claude mcp add freedcamp -s user \
    -e FREEDCAMP_API_KEY=… -e FREEDCAMP_API_SECRET=… \
    -- node dist/server.js

# prefer secrets out of the config? pull them from AWS SSM at startup
$ claude mcp add freedcamp -s user -e AWS_PROFILE=… -- node dist/ssm-launch.js

# ✓ Status: Connected — 12 tools ready

Questions

Freedcamp MCP, answered.

What is the Freedcamp MCP server?
It's an open-source Model Context Protocol (MCP) server that connects AI assistants like Claude to Freedcamp, with full read and write access across every project — so Claude can manage tasks, projects and comments directly inside your workspace.
What can AI do with Freedcamp through this MCP?
Twelve tools: discover projects, task lists and users; create, list, update and delete tasks; create and update projects (including archive / unarchive); and add, edit and delete comments. Everything supports bulk operations and works across all your projects — not just one.
How is this different from the original gabeosx Freedcamp MCP?
Upstream was locked to a single project and tasks-only. This fork removes the lock (a per-call project_id always wins), and adds discovery tools, project write tools, comment tools, pagination-aware listing, per-item error isolation in bulk operations, an AWS SSM credential launcher and optional bearer-token auth on the HTTP transport.
How do I install the Freedcamp MCP with Claude Code?
It installs from source (it's a fork, so npx freedcamp-mcp would fetch the upstream). Clone the repo, run npm install && npm run build, then claude mcp add freedcamp -s user -e FREEDCAMP_API_KEY=… -e FREEDCAMP_API_SECRET=… -- node dist/server.js. Both paths are in the install section above.
Is the Freedcamp MCP server free and open source?
Yes — it's MIT-licensed and free to use, forked from gabeosx/freedmcpcamp and extended with multi-project support plus the project and comment tools.
Where do the Freedcamp credentials live?
You bring a Freedcamp API key + secret (Freedcamp → avatar → Settings → API). Pass them as environment variables, or store them in AWS SSM Parameter Store so your config references only the parameter names — never the secret values. The optional HTTP transport can also require a bearer token.
